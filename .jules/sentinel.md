## 2025-02-14 - Fix Command Injection in MCP Server tool call handling
**Vulnerability:** Command injection vulnerability in `Library/Homebrew/mcp_server.rb` where arguments passed via JSON-RPC to MCP Server tools were incorrectly escaped using `Shellwords.escape(arg)` before passing an array to `Open3.popen2e`. `Open3.popen2e` treats an array as an unescaped set of args but `Shellwords.escape` forces it into a single shell string, which allowed execution of shell commands when passed an array with multiple words.
**Learning:** `Open3.popen2e` and similar methods automatically handle argument escaping when passed an array (unlike a single string). Using `Shellwords.escape` on an array element actually bypasses `Open3`'s escaping and treats the entire block as raw input for `sh -c` under the hood if it contains shell metacharacters, potentially leading to command injection or at minimum broken arguments.
**Prevention:** Do not use `Shellwords.escape` when passing an array of arguments to `Open3.popen2e`, `system`, or `exec`. Let the array format naturally prevent shell injection.
